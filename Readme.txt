【対象日時】2024年10月13日10時00分
【対象系統】北海道系統、187kV以上の基幹系統のループ部分
【発電機数】29機＋スラックノード1個所＝30か所
【ノード数】71
【送電線数】90


# 発電機データフォーマット
【gen_name】発電機名称。
【gen_type】発電機種別。
【bus】発電機が接続されているノードID。
【Pg】発電機の有効電力出力 [MW]。対象ノードがPV指定ノードの場合、有効電力の指定値に相当。スラックノードの場合、潮流計算によって得られた発電機の有効電力の計算結果に相当。
【Qg】発電機の無効電力出力 [MVar]。潮流計算によって得られた発電機の無効電力の計算結果に相当。
【Vg】発電機の指定電圧 [p.u.]。


# ノードデータフォーマット
【bus_name】ノード名称。
【bus_i】ノードID。
【type】ノードの種別。1⇒PQ指定ノード、2⇒PV指定ノード、3⇒スラックノード
【Pd】有効電力負荷 [MW]。今回は再エネもマイナスの負荷として各ノードの負荷に合算。
【Qd】無効電力負荷 [MVar]。今回はゼロ。
【Vm】各ノードの電圧の大きさ [p.u.]。
【Va】各ノードの電圧位相 [°]。
【baseKV】各ノードの電圧階級 [kV]。


# 送電線データフォーマット
【branch_name】送電線名称。
【fbus】送電端のノードID。
【tbus】受電端のノードID。
【r】送電線の抵抗値 [%（1000MVAベース）]。回線数を反映済み。
【X】送電線のリアクタンス [%（1000MVAベース）]。回線数を反映済み。
【b】送電線のサセプタンス [%（1000MVAベース）]。回線数を反映済み。
【P_f】送電端から送られる有効電力 [MW]。または、送電端が受け取る有効電力にマイナスをかけた値。
【Q_f】送電端から送られる無効電力 [MVar]。または、送電端が受け取る無効電力にマイナスをかけた値。
【P_t】受電端から送られる有効電力 [MW]。または、受電端が受け取る有効電力にマイナスをかけた値。
【Q_t】受電端から送られる無効電力 [MVar]。または、受電端が受け取る無効電力にマイナスをかけた値。
